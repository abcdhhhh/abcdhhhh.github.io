2. 处理器体系结构

LC-2200：

  1. 8 条指令 。一条指令只完成一个操作（add, nand, addi, lw, sw, beq, jalr, halt）
  2. 指令长度固定 。 32 位
  3. 只有 Load 和 Store 指令可以访问存储器 。
  4. 寻址模式简单 。简化了从存储器取数操作数

寻址模式：

  1. 寄存器寻址
  2. 立即数寻址
  3. 内存寻址：基址+偏移量
  4. if-then-else寻址：PC+偏移量
  5. switch寻址：
    1. 有限、稀疏：用多个if-then-else
    2. 连续、不稀疏：跳转表

编译一个过程调用的步骤：

  1. 保证**调用者的状态（寄存器）**被保护好以便从过程调用中返回时得以恢复
  2. 将实际参数传递给调用者：寄存器（少量参数）+内存栈（多余参数）
  3. 记住返回地址：JAL ，保存返回地址，PC=被调用者起始地址
  4. 将控制权交给被调用者
  5. 为被调用者的局部变量分配空间：内存栈
  6. 从被调用者接受返回值并传给调用者：寄存器（少量参数）+内存栈（多余参数）
  7. 返回调用点：JAL ，PC=返回地址

保存策略：谁使用公共寄存器，谁负责保存和恢复

栈的状态：

  1. 调用者保存t0~t2（临时）
  2. 调用者在a0~a2放置参数，保存额外参数
  3. 调用者保存额外返回值
  4. 调用者保存**$ra**（上层返回地址）
  5. 调用（$ra记录返回地址，PC跳转）
  6. 被调用者保存***$fp***（上层帧指针），并将$sp赋值给$fp
  7. 被调用者保存***s0~s2***（调用者的寄存器）
  8. 被调用者给***局部变量***分配栈空间
  9. 返回（PC跳转）
  10. 调用者恢复$ra
  11. 调用者根据需要恢复额外返回值
  12. 调用者丢弃额外参数
  13. 调用者恢复t0~t2

变长指令：不浪费空间，没有操作数位数限制，指令集设计更灵活；实现更加复杂

3. 处理器实现

宏状态：FETCH, DECODE, EXECUTE

微状态：ifetch1, ifetch2, …

  1. FETCH
    1. ifetch1: PC→MAR,PC→APC \to MAR, PC\to APC→MAR,PC→A
    2. ifetch2: MEM[MAR]→IRMEM[MAR]\to IRMEM[MAR]→IR
    3. ifetch3: A+1→PCA+1\to PCA+1→PC
  2. DECODE
    1. beq1: Ry→ARy\to ARy→A
    2. beq2: Rz→BRz\to BRz→B
    3. beq3: A−B→ZA-B\to ZA−B→Z
    4. beq4: PC+1→APC+1\to APC+1→A
    5. beq5: Ext(offset)→BExt(offset)\to BExt(offset)→B
    6. beq6: A+B→PCA+B\to PCA+B→PC
  3. EXECUTE

总线

串行：成本低，节省布线空间，适合长距离传输，工作频率高；需串-并行转换

并行：时序逻辑简单；远距离成本高，占用更多空间，无法持续提高工作频率（信号线间干扰）

单总线：结构简单，成本低，易于接入新设备；多个部件争用，带宽低，负载重，不支持并发传送

双总线：提高系统吞吐量，支持突发传送；需增加通道硬件设备、额外的管理

三总线：提高高速外设访问速度；任一时刻只能使用一种总线，系统工作效率较低

四总线：桥

总线仲裁：多个主设备竞争

4. 中断、陷入及异常

程序不连续性：

  • 中断：外部设备引起的不连续性

  • 陷入：当前程序无法处理，只能由OS处理

  • 异常：脱离程序正常执行轨迹，由当前程序处理

体系结构与操作系统的合作：

  1. CPU 定义中断、异常及其编号向量值
  2. OS 定义异常软件中断、陷入系统调用及其编号向量值
  3. 在启动时，OS建立中断向量表IVT，给出处理过程的地址
  4. 在普通程序执行时，硬件检测异常和陷入，并将向量值 → ETR 寄存器
  5. 在普通程序执行时，硬件检测外部中断 ，接收中断设备的向量值
  6. 硬件使用向量值索引中断向量表 ，将控制权交给处理过程
  1. 保存 $k0
  2. 打开中断
  3. 保存寄存器
  4. 执行设备代码
  5. 恢复寄存器
  6. 关闭中断
  7. 恢复 $k0
  8. 返回原始程序
    1. 原子操作：PC <- $k0 ，打开中断

处理器与设备之间的信号交换：

  1. 设备有中断时，对 INT 线 断言
  2. 处理器在完成当前指令后，检查 INT 线，判断是否有中断请求
  3. 若有中断，则处理器进入 INT 宏状态，并将总线上的 INTA 线 断言
  4. 设备收到处理器发出的 INTA 信号后，将自己的中断向量放到数据总线
  5. 处理器收到中断向量后，在 中断向量表 中查出处理过程的地址
  6. 将 PC 保存到 $k0 ，并将中断向量表中取出的值填入 PC

5. 处理器性能与流水线处理器的设计

SPEC基准测试

SPECratio=参考机器执行时间目标机器执行时间SPEC_{ratio}=\frac{参考机器执行时间}{目标机器执行时间}SPECratio​=目标机器执行时间参考机器执行时间​

响应时间（墙上时间）：计算机完成某任务的总时间

吞吐率（带宽）：单位时间内完成的任务数量

存储器容量：所有存储单元的总数目

基本原则：大概率事件优先，Amdahl定律（加快某部件执行速度所获得的系统性能加速比受限于该部件在系统中所占的重要性比例）

波特率：全算

比特率：只算有效信息

7. 内存管理技术

基址+限长寄存器：方便进行动态重定位

固定尺寸分区：有内部、外部碎片

​ 分配表以固定尺寸分区为基础维护着总结信息

变长分区：有外部碎片，可以缩并

分页：可能有内部碎片

分段：每段地址都从0开始，可能有外部碎片

9. 分级存储体系

缓存缺失类型：

  1. 冷/强制缺失：第一次访问给定内存单元
  2. 容量缺失：缓存是满的
  3. 冲突缺失：缓存不满，但映射策略会引入被其他元素占用的缓存行
